# GeminiClaw vs OpenClaw Base (Pi Agent 비교 분석서)

> **작성일**: 2026년 2월 25일  
> **프로젝트**: GeminiClaw  
> **작성 목적**: 기존 OpenClaw 프로젝트의 `pi_agent` 대비 현재 GeminiClaw에 포팅된 `Pi Agent`의 기능 구현 수준 및 고도화 차이점 문서화

---

## 1. 핵심 아키텍처 및 역할 (완전 구현 + 확장)

*   **Base OpenClaw**: 단일 AI(Pi)가 로컬 컴퓨터 호스트 제어 권한을 가지고 터미널 CLI 상향에서 동작하는 독립적인 1:1 대화형 비서.
*   **GeminiClaw (현재)**: 
    *   Base 모델의 1:1 독립형 아키텍처를 완벽하게 유지(`pi_agent.py`). 
    *   Team OS(다중 에이전트 라우팅) 로직과 시각적으로 분리된 **Dual-Mode Flutter UI**를 통해 유저 입장에서 독립된 공간으로 완벽히 분리. (UI 좌측 사이드바 구조 도입)
    *   **결론**: 포팅 완료 및 GUI 환경으로의 구조적 진화 완료.

## 2. 권한 및 도구 (Tools) 통합 수준 (탁월한 제어력 및 보안)

### 2.1 로컬 파일 제어 (`read_file`, `write_file`)
*   **Base**: 지원
*   **Current**: 완벽 지원. 안전하게 로컬 파일(소스코드, 문서 등)을 읽고 쓸 수 있음.

### 2.2 터미널 시스템 제어 (`execute_shell_command`)
*   **Base**: 호스트 PC의 터미널 명령어 직접 실행 권한 보유 (위험성 내포).
*   **Current**: 완벽 지원 및 보안 아키텍처 도입. 
    *   보안 페이즈(Phase 4)를 거치면서 일반 워커 에이전트들은 Docker 샌드박스로 격리된 터미널에서 실행됨. 
    *   반면 **Pi Agent는 CTO 직속 비서이므로 신뢰 모델 기반 호스트(Host) 실행 권한을 유지**하여 Base 버전과 동일한 막강한 편의성 보장.
    *   동시에 시스템 위험 명령어에 대해서는 **HITL (Human-in-the-Loop) 승인 팝업 시스템**을 도입하여 보안 밸런스를 크게 높임.

### 2.3 Mac 전용 네이티브 연동 (`send_notification`, `clipboard`)
*   **Base**: AppleScript 기반 알림 전송 및 클립보드 I/O.
*   **Current**: `tools/system_langchain.py`를 통해 동일 기능을 완벽히 포팅. 실제 UI 테스트 환경에서 macOS 알림 발송 및 클립보드 내용 읽기/쓰기가 100% 동일하게 호환 작동함을 검증 완료.

## 3. 상태 관리 및 장기 기억 모델 (기존 한계 완벽 극복)

*   **Base OpenClaw**: 
    *   단일 터미널 Python 세션 실행 구조.
    *   스크립트(또는 챗)가 종료되면 이전 대화의 문맥(Context)과 작업 메모리가 완전히 유실되는 치명적 한계 존재.
*   **GeminiClaw (현재)**: 
    *   Phase 0 및 Phase 2 단계를 통해 **`core/memory.py` 어댑터와 `LangGraph SqliteSaver`를 결합** 적용.
    *   브라우저를 껐다 켜거나 백엔드 서버를 완전히 재시작해도, UUID 세션 기반으로 **Pi Agent와의 채팅 기록과 문맥이 영구적으로 보존(Persistent Memory)** 되도록 상태 관리 아키텍처를 완전히 갈아엎고 재구축.

## 4. 특별 독점 기능: 자가 발전 (Self-Evolution)

*   **해당 기능은 Base OpenClaw에는 존재하지 않는 완전한 독자적 오리지널 엔지니어링 결과물**입니다.
*   **기능명**: `update_skill_manual` (Phase 5.4 완료)
*   **상세 설명**: 
    *   Pi Agent는 단순한 비서를 넘어 회사의 '운영 옵저버/관리자'로 격상됨.
    *   CTO(유저)가 Pi Agent에게 피드백을 전달하면("ex: 마케터 업무 매뉴얼에 ASO 전략 추가해"), Pi Agent가 내부 도구를 호출하여.
    *   다른 일반 실무 에이전트들(PM, HR, 개발자, 마케터 등)의 **업무 지침서(Markdown)를 직접 열어 LLM이 매끄럽게 구조를 해치지 않으면서 규칙을 덮어씁니다.**
    *   이를 통해 시스템 코드를 건드리지 않고 자연어 채팅만으로 회사 운영 규칙을 자체 갱신해나가는 자가 진화 능력을 갖춤.

---

## 최종 평가

현재 GeminiClaw에 이식된 **Pi Agent는 Base 버전(OpenClaw)이 최초에 의도했던 '독립된 개인 비서 역할, 파일 I/O 및 쉘 최고 권한 설정, Mac 클립보드/알림 시스템 연동'이라는 3대 핵심 목표를 100% 포팅 완료**하였습니다.

나아가 Base 버전에 없었던 **웹 UI 소켓 스트리밍, SQLite 기반 스레드 영구 기억 체계, 위험 명령어 승인(HITL) 체제 도입**, 그리고 무엇보다 **조직 내 타 에이전트의 스킬 지침서를 직접 수정하는 "Self-Evolution(자가 발전)" 기능**까지 탑재함으로써 단순 포팅을 넘어선 **기반 코드를 초월한 상위 확장 버전(Super-Set)**으로 완성도 높게 구축되었습니다.
